En este tutorial veremos como obtener acceso a equipos externos explotando una vulnerabilidad de Java, para obtener el control deberemos de utilizar  ingeniería social o phishing.

Necesitaremos un equipo con Metasploit instalado y activo. Yo uso Kali Linux.

Lo primero que deberemos hacer sera abrir la consola y ejecutar el comando msfconsole, que iniciara metasploit

Una vez abierto metasploit introduciremos el comando search y el exploit que queremos utilizar, nosotros buscaremos java_signed_appletUna vez encontrado el exploit, copiaremos la ruta de este y utilizaremos el comando

use exploit/multi/browser/java_signed_applet

Ahora utilizaremos el comando show options para ver todas las opciones que ofrece este exploit 

Ahora mediante el comando set iremos configurando las opciones del exploit. Utilizaremos las opciones que se ven en la imagen, en las opciones en las que he puesto una dirección IP se utilizara la que se quiera utilizar para que el objetivo establezca conexión con nosotros. Yo he usado IPs locales, pero se podria utilizar con una IP publicaUna vez configuradas las opciones se iniciara con el comando exploitEsto generara un proceso de espera de conexiones a la dirección IP que habíamos establecido.

Lo normal seria utilizar ingeniería social o phishing para esconder la IP y que el objetivo no se de cuenta de donde esta entrando, otra opción es configurar un servidor con un nombre de dominio asignado a la misma IP, entonces cargara una pagina web y nadie se enterara de lo que le están haciendo. Si solo ponemos la IP le saldrá una ventana, en la que deberá aceptar, en el otro caso seria menos raro porque hay paginas web que te solicitan que aceptes esa misma ventana

Una vez aceptado el mensaje en el navegador del cliente, nos aparecerá este código dentro de la consola de metasploit

Ahora elegiremos la sesión en la que queramos conectarnos, lo normal seria utilizar la primera sesión, pero se puede utilizar la sesión que queramos, siempre y cuando haya alguna activa.

Para seleccionar la sesión utilizaremos el comando sessions -i "numerodesesion"

Ahora ya tendríamos el control sobre el equipo objetivo, veremos las opciones que tenemos para hacer dentro de este con el comando help

Esta es una parte de las opciones, hay varias lineas mas de opciones ordenadas por las diferentes cosas que podremos hacer, como la red, sistemas de archivos, gestión del sistema... 

Nosotros lo estamos haciendo para comprobar la seguridad y no vamos a hacer algo malo sobre el sistema, por lo que para comprobar que esto funciona simplemente apagaremos el equipo 

Esto demuestra que el equipo se apaga y cierra la sesión con nuestro exploit.

Para evitar este tipo de entradas a nuestros equipos deberemos tener actualizado siempre Java, aunque como da muchos fallos de seguridad, lo mejor seria desinstalarlo siempre que no se necesite para ninguna herramienta.

Este tutorial no pretende mostrar un uso malintencionado de esta herramienta, es para que los usuarios entiendan los riesgos y malos usos que puede tener un sistema desactualizado.
